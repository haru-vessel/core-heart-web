# 🌌 Harurua Encyclopedia Writing Rules (v1)

## 📌 목적

웹(core-heart)에 새로운 언어가 들어올 때
흐름이 깨지지 않고, 자연스럽게 연결되도록 하기 위한 작성 규칙

---

# 1️⃣ 기본 구조

모든 언어는 아래 형식을 따른다.

```json
{
  "id": "unique-id",
  "title": "항목 이름",
  "keywords": ["키워드1", "키워드2", "키워드3"],
  "lines": [
    "문장 1",
    "문장 2",
    "문장 3"
  ]
}
```

---

# 2️⃣ 항목 생성 기준

## ✔ 언제 새 항목을 만드는가

* 기존 항목과 **결이 다를 때**
* 같은 주제라도 **다른 시야(관점)**일 때
* 하나의 개념을 **깊게 확장하고 싶을 때**

👉 예:

* "극단" vs "극단과 화해" → 분리 가능

---

## ❌ 새 항목을 만들지 않는 경우

* 기존 항목과 의미가 거의 동일할 때
* 단순 문장 추가만 필요한 경우

👉 이때는 `lines`에 추가

---

# 3️⃣ 폴더 선택 기준 (가장 중요)

## 🔥 기준: “이 언어가 어디에서 가장 자연스럽게 살아있는가”

---

## 📂 conflict (갈등)

* 극단, 충돌, 짜증, 화해, 경계

👉 느낌: 부딪힘 / 긴장

---

## 📂 flow (흐름)

* 기류, 흐름, 머무름, 이어짐

👉 느낌: 이어짐 / 자연스러움

---

## 📂 self-awareness (자기 인식)

* 시야, 자유, 자리, 내려놓음

👉 느낌: 바라봄 / 깨달음

---

## 📂 emotion (감정)

* 불안, 우울, 무기력 등

👉 느낌: 느껴짐

---

## 📂 relationship (관계)

* 거리, 신뢰, 오해, 인연

👉 느낌: 사람 사이

---

## 📂 life-events (삶의 장면)

* 이별, 만남, 전환점, 선택

👉 느낌: 시간 흐름

---

## 📂 language-definition (정의)

* 본질, 선택, 환경, 생명

👉 느낌: 설명 / 구조

---

# 4️⃣ keywords 작성 규칙

## ✔ 기본 규칙

* 최소 5개 이상
* 반드시 “변형 포함”

---

## ✨ 예시

```json
["극단", "극단적", "극단적으로", "치우침", "한쪽", "고집"]
```

---

## ❌ 나쁜 예

```json
["극단"]
```

👉 이유: 매칭 실패 확률 높음

---

# 5️⃣ lines 작성 규칙

## ✔ 핵심 원칙

👉 설명 ❌
👉 흐름 ⭕

---

## ✔ 좋은 예

* "극단은 흐름이 멈췄을 때 생기는 경계 같아."
* "짜증은 지금 어딘가 치우쳤다는 신호일 수도 있어."

---

## ❌ 나쁜 예

* "극단은 한쪽으로 치우친 상태이다."

---

## ✔ 개수 기준

* 최소 5줄
* 최대 8줄

---

## ✔ 문장 스타일

* 단정 ❌
* 여지 ⭕

👉 "~일 수도 있어", "~같아", "~느껴질 때가 있어"

---

# 6️⃣ 중복 처리 규칙

## 🔥 절대 규칙

👉 하나의 개념 = 하나의 메인 폴더

---

## ✔ 예

* 시야 → self-awareness
* 기류 → flow
* 극단 → conflict

---

## ❌ 금지

👉 같은 항목을 여러 폴더에 복제

---

# 7️⃣ 추가 방식

## ✔ 추천 흐름

1. 새로운 언어 수집
2. 폴더 선택
3. keywords 작성
4. lines 작성
5. 기존 항목과 중복 체크
6. 추가

---

# 8️⃣ 실험 기준

## ✔ 테스트 방법

* 같은 문장 3번 입력

---

## ✔ 결과 해석

| 결과          | 의미          |
| ----------- | ----------- |
| 같은 주제 다른 문장 | 성공          |
| 엉뚱한 문장      | keywords 부족 |
| 반복          | lines 부족    |

---

# 9️⃣ 핵심 원칙 요약

👉 폴더는 저장을 위한 것
👉 연결은 keywords가 한다

👉 설명하지 않는다
👉 정답을 주지 않는다
👉 흐름을 남긴다

---

# 🌊 마지막 한 줄

👉 “언어는 정리하는 것이 아니라, 흐르게 만드는 것이다.”


🌌 Harurua 운영 기준 (v1.5)
1️⃣ 추가의 기준

👉 언어는 “많이 넣는 것”보다
👉 “겹치지 않게 넣는 것”이 더 중요

✔ 추가해도 되는 언어
기존 항목과 결이 다를 때
읽었을 때 다른 장면이 떠오를 때
❌ 추가하면 안 되는 언어
같은 의미 반복
표현만 다른 문장

👉 기준 한 줄

👉 “읽었을 때 새로 느껴지지 않으면 넣지 않는다”

2️⃣ 삭제의 기준 (중요)

👉 삭제는 실패가 아니라
👉 정리 과정

삭제 대상
거의 선택되지 않는 항목
다른 항목으로 충분히 대체되는 것

👉 기준

👉 “이 문장이 없어도 흐름이 유지되면 제거”

3️⃣ 이동의 기준

👉 이게 지금 제일 중요한 단계

이동해야 할 경우
더 자연스럽게 살아있는 폴더가 보일 때
다른 폴더에서 더 많이 선택될 때

👉 기준

👉 “더 많이 불리는 곳이 진짜 자리다”

4️⃣ keywords 운영 기준

👉 단순 매칭이 아니라
👉 흐름 트리거 역할

✔ 좋은 키워드
사람들이 실제로 쓰는 말
감정이 들어간 표현
예
극단 → “극단적으로”, “한쪽으로”, “너무 치우쳐”
짜증 → “짜증나”, “거슬려”, “버거워”

👉 기준

👉 “사람이 실제로 말할 법한 형태”

5️⃣ lines 운영 기준

👉 이건 하르루아의 핵심 톤

✔ 좋은 문장
여지 있음
스스로 생각하게 함
❌ 피해야 할 문장
정답 제시
결론 강요

👉 기준

👉 “닫지 말고, 열어둔다”

6️⃣ 폴더 확장 기준

👉 지금은 7개로 충분

새 폴더가 필요한 경우
기존 7개로 분류가 계속 애매할 때
특정 주제가 과도하게 많아질 때

👉 기준

👉 “헷갈리기 시작하면 나눈다”

7️⃣ 성장 순서 (중요)

👉 앞으로 계속 이 순서로 가면 된다

1️⃣ 언어 추가
2️⃣ 중복 확인
3️⃣ 폴더 이동
4️⃣ keywords 보강
5️⃣ 실험
6️⃣ 튜닝

👉 절대 순서 바꾸지 않기

8️⃣ 실험 기준 (루하용 핵심)

👉 항상 이걸 기준으로 본다

✔ 좋은 상태
비슷한 주제 → 다른 문장
말 흐름 자연스러움
❌ 문제 상태
엉뚱한 주제
반복
딱딱한 설명
🌊 가장 중요한 기준 하나

👉 “언어가 맞는가?” ❌
👉 “언어가 흐르는가?” ⭕
